1️⃣ 章節 1 · 準備題庫 · Step 1

Step 1：把題目 PDF 丟進 NotebookLM

1

打開 notebooklm.google.com → 用 Google 帳號登入

2

點左上角 「+ 建立新筆記本」（或 + New notebook）

3

把你的題目 PDF 檔拖進中央上傳區（或點「上傳」選檔案）

✅ 支援格式：PDF / Word / Google Docs / 純文字 / 網址

✅ 一次可以丟多個檔案（同單元 / 同章節題目都丟進去）

4

等個 10~30 秒，NBLM 會跑出「來源摘要」→ 表示讀好了 ✓

💡 小撇步：題庫如果是圖片掃描（如手寫考卷拍照），NBLM 也能讀。但建議先用手機掃描 App 轉成 PDF，認字率會更高。

2️⃣ 章節 1 · 準備題庫 · Step 2

Step 2：下提示詞，讓 NBLM 整理成標準表格

把下面這段 prompt 貼進 NotebookLM 下方的對話框，按送出。

請幫我把這份題庫整理成「四選一選擇題」的標準表格。 格式要求： 1. 每一列就是一題 2. 欄位順序：題目 | A | B | C | D（總共 5 欄） 3. A 欄位「一律放正確答案」 （不管原本標的是哪個選項，正解都搬到 A） 4. B、C、D 欄位放剩下 3 個錯誤選項 5. 不要包含題號、不要分節、不要其他說明 6. 如果某題未滿 4 個選項，缺的格子用「空」補上 7. 用 Markdown 表格格式輸出，方便我複製貼到 Excel

✅ 完成的表格會長這樣：

題目A（正解）BCD
蘋果的英文applebananaorangegrape
世界最高山珠穆朗瑪峰玉山富士山阿爾卑斯山

⚠️ 檢查重點：A 欄是不是全部都是正確答案？沒有就再叫 NBLM 重整：「A 欄要放正解，請重新整理」。

3️⃣ 章節 1 · 準備題庫 · Step 3

Step 3：存成記事 → 匯出為試算表

NotebookLM 內建匯出功能，2 步搞定。

1

NBLM 跑出表格後，在那段回應的下方點 「儲存到筆記」（或圖示像便利貼的按鈕）

→ 右邊側欄會出現一筆新的「記事」

2

點那筆記事右邊的 「⋮」三點選單 → 選 「匯出為試算表」

→ NBLM 會自動建一份 Google Sheets，並直接打開

3

確認 Sheets 內容無誤後，重新命名：【科目】題庫_v1

例：「國一英文_單字題庫_v1」

4

需要 Excel 檔的話：檔案 → 下載 → Microsoft Excel (.xlsx)

不需要的話就留在 Google Sheets 也 OK（下一節要用 Sheets 做活的網址）

✅ 到這邊，題庫已經在 Google Sheets 裡了。
下一步把它變成遊戲能讀的活網址。

4️⃣ 章節 1 · 準備題庫 · Step 4 ⭐

Step 4：把 Sheets 發布成 CSV，拿到活網址

這是整個課程的魔法時刻——一條網址，遊戲跟著題庫一起更新。

1

在 Google Sheets 右上角點 「檔案」 → 「共用」 → 「發布到網路」

2

跳出視窗後，2 個下拉選單要選對：

· 左邊：整份文件（或「Sheet1」也行）

· 右邊：逗號分隔值 (.csv)　⚠️ 不要選網頁、不要選 Excel

3

點藍色 「發布」 按鈕 → 跳確認 → 點「確定」

4

會跳出一條網址（很長一串，docs.google.com/...）→ 複製起來

這條網址 = 你的「活題庫」，遊戲會用它讀題目

5

驗證能用：把網址貼到瀏覽器網址列按 Enter

→ 應該會下載一個 .csv 檔，或直接顯示「題目,A,B,C,D」這樣的內容 = 成功 ✓

💡 之後你想改題目？

直接在 Google Sheets 改 → 存檔 → 遊戲自動更新，不用重做網頁。

成語填字 — 拖字卡補空格

玩法：上方成語缺 2 字，下方 15 字卡拖曳配對。國文老師複習成語最快上手。

🔗 開啟示範作品
我是國中國文老師，要做一個「成語填字」拖曳遊戲給學生玩。 請在我目前 IDE 打開的「我的小遊戲」資料夾裡，建一個叫「示範2_成語填字」的子資料夾，做一個 index.html。 ═══ 題庫資料來源 ═══ 不用外部 CSV，題目直接硬寫在 JS 裡，至少 10 題。 每題格式： const QUESTIONS = [ { 成語: '一鳴驚人', 缺字: [1, 2], 答案: ['鳴', '驚'], 意思: '比喻平時默默無聞，突然有驚人表現。' }, { 成語: '風花雪月', 缺字: [1, 3], 答案: ['花', '月'], 意思: '原指四時景色，後比喻浮華空洞的詩文。' }, { 成語: '三番五次', 缺字: [1, 3], 答案: ['番', '次'], 意思: '形容次數很多。' }, { 成語: '一字千金', 缺字: [1, 3], 答案: ['字', '金'], 意思: '形容文字價值極高。' }, { 成語: '畫龍點睛', 缺字: [1, 3], 答案: ['龍', '睛'], 意思: '比喻在關鍵處點明要旨。' }, // …請幫我補滿至少 10 題，挑國中常見成語 ]; 每題的「缺字」是索引（0-3），代表要挖空的位置；答案是按缺字順序對應的字。 ═══ 遊戲核心玩法 ═══ 1. 一場遊戲共 5 題（從題庫隨機抽，不重複） 2. 每題畫面： - 上方大字顯示成語，缺字位置用方框「□」代替 - 例：「一 □ 千 □」 - 下方排 15 張字卡（隨機排列）：2 個正解 + 13 個干擾字 - 字卡順序每次都打亂 3. 玩家把字卡用拖曳拉到上方的空格 4. 判定： - 拖對 → 字卡變綠、鎖定在空格內、播清脆鈴聲 - 拖錯 → 字卡彈回原位、紅光閃 0.2 秒 5. 兩個空格都填對 → 顯示「✓ 答對！」+ 成語意思 + 下一題按鈕 6. 5 題完成 → 顯示總成績 + 重新挑戰按鈕 ═══ 視覺風格 ═══ - 古色古香、書卷字帖風 - 背景：米黃宣紙色 + 淡淡紋理 - 標題與成語：毛筆楷書字型（用「Noto Serif TC」） - 空格用紅色方框（像書法字帖的米字格） - 字卡用淺色木紋或淡黃底 + 黑色楷體 - 答對時空格內字變為紅色（像紅章蓋上去） ═══ 技術需求 ═══ - 單一 index.html（CSS + JS 全部塞進去） - 不依賴第三方套件（字型用 Google Fonts CDN 可以） - 中文 UTF-8 編碼 - 拖曳同時支援桌機滑鼠 + 平板觸控 完成後請用本地瀏覽器幫我打開。

✅ 章節 11 · 補充 ⑤ · 什麼時候用

什麼時候叫子模型？什麼時候自己做？

✅ 適合用子模型

  • 大量讀檔但只要結論
    例：把 50 份合約丟一隻去摘要
  • 多個獨立查詢同時跑
    例：A 找客戶資料、B 找上月發票
  • 需要獨立第二意見
    例：另開一隻 review 主 AI 寫的東西
  • 背景跑長任務
    你繼續做別的事，它跑好通知你

❌ 不適合用

  • 對話脈絡複雜
    子模型不知道前後文，要花成本重講
  • 步驟有先後依賴
    本來就要 A 做完才能做 B
  • 小任務
    自己順手做更快
  • 需要你即時參與
    子模型獨立跑，你看不到中間過程

怎麼叫 AI 開子模型？

不用記指令，用自然語言就好——關鍵字是「同時」「平行」「分頭」。

幫我同時處理這 20 個客戶的月結， 每個客戶開一隻子模型去做， 最後把所有結果彙整成一張總表給我。

✨ 重點 1：說清楚邊界

每隻子模型負責「一個客戶」，不要互相干擾。

✨ 重點 2：指定彙整方式

不講的話子模型結果會四散，要明說「最後合成一份」。

學會這招 = AI 從「個人助理」變「整個團隊」🚀
