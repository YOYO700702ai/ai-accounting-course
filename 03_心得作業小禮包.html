朝代配對 — 古今對對碰
我是國中歷史老師，要做一個「朝代配對」練習網頁給學生玩。
請在我目前 IDE 打開的「我的小遊戲」資料夾裡，建一個叫「示範1_朝代配對」的子資料夾，做一個 index.html。

═══ 題目資料（直接寫在 JS 裡，不用外部檔案）═══
請在 JS 裡建兩個陣列：
- 6 個朝代（每個朝代有 id 跟名稱）
- 15+ 個事件（每個事件有 dynasty_id 對應到朝代，可以多對一）
- 每個朝代可以對應 1 個以上的事件（數量不一致沒關係）

朝代陣列（左欄）：
  { id: 1, name: "秦" }
  { id: 2, name: "漢" }
  { id: 3, name: "唐" }
  { id: 4, name: "宋" }
  { id: 5, name: "明" }
  { id: 6, name: "清" }

事件陣列（右欄，至少 15 個）：
  { id: 'e1',  dynasty_id: 1, text: "統一六國",     icon: "⚔️" }
  { id: 'e2',  dynasty_id: 1, text: "修築萬里長城", icon: "🧱" }
  { id: 'e3',  dynasty_id: 1, text: "焚書坑儒",     icon: "🔥" }
  { id: 'e4',  dynasty_id: 2, text: "張騫通西域",   icon: "🐪" }
  { id: 'e5',  dynasty_id: 2, text: "文景之治",     icon: "📜" }
  { id: 'e6',  dynasty_id: 2, text: "罷黜百家獨尊儒術", icon: "📖" }
  { id: 'e7',  dynasty_id: 3, text: "貞觀之治",     icon: "👑" }
  { id: 'e8',  dynasty_id: 3, text: "安史之亂",     icon: "💥" }
  { id: 'e9',  dynasty_id: 3, text: "玄奘西行取經", icon: "🧘" }
  { id: 'e10', dynasty_id: 4, text: "王安石變法",   icon: "📜" }
  { id: 'e11', dynasty_id: 4, text: "岳飛抗金",     icon: "🗡️" }
  { id: 'e12', dynasty_id: 5, text: "鄭和下西洋",   icon: "⛵" }
  { id: 'e13', dynasty_id: 5, text: "永樂大典編成", icon: "📚" }
  { id: 'e14', dynasty_id: 6, text: "鴉片戰爭",     icon: "💣" }
  { id: 'e15', dynasty_id: 6, text: "康熙盛世",     icon: "🌅" }
  { id: 'e16', dynasty_id: 6, text: "辛亥革命終結", icon: "🏛️" }

═══ 遊戲核心玩法 ═══
1. 畫面分左右兩區
   - 左欄：6 個朝代「卡格」（朝代名在上方，下方是「掉落區」可放多張事件卡）
   - 右欄：15+ 張事件卡，順序隨機打亂
2. 玩家從右欄把事件卡拖曳到左欄對應的朝代格子
3. 用 HTML5 drag-and-drop API
4. 配對判斷：
   - 正確（dynasty_id 相符）→ 事件卡變綠、鎖定在朝代格內（堆疊顯示）
   - 錯誤 → 紅色閃光 0.3 秒 + 卡片彈回右欄原位
5. 每次嘗試（不論對錯）都計數
6. 右欄事件卡全部正確放完 → 過關畫面（金色閃光 + 顯示總嘗試次數 + 再玩一次按鈕）

═══ 視覺風格 ═══
- 古典中國風：深色底 + 金色邊框 + 紅色點綴
- 朝代卡：金色邊框、米黃底、朝代名用大字
- 事件卡：紅色邊框、深色底、icon + 事件名稱
- 已配對的事件卡用較小縮圖形式堆疊在朝代格內
- 字型「Noto Serif TC」古典襯線
- RWD：手機改上下兩欄；事件卡區可橫向捲動

═══ 技術需求 ═══
- 單一 index.html（CSS + JS 全部塞在一個檔案裡）
- 不依賴任何第三方套件
- 中文 UTF-8 編碼
- 題目資料直接寫在 JS 裡
- 朝代格的掉落區要能容納多張事件卡

📝 章節 6 · 示範 2 · 成語填字
成語填字 — 拖字卡補空格
我是國中國文老師，要做一個「成語填字」拖曳遊戲給學生玩。
請在我目前 IDE 打開的「我的小遊戲」資料夾裡，建一個叫「示範2_成語填字」的子資料夾，做一個 index.html。

═══ 題庫資料來源 ═══
不用外部 CSV，題目直接硬寫在 JS 裡，至少 10 題。
每題格式：

const QUESTIONS = [
  { 成語: '一鳴驚人', 缺字: [1, 2], 答案: ['鳴', '驚'], 意思: '比喻平時默默無聞，突然有驚人表現。' },
  { 成語: '風花雪月', 缺字: [1, 3], 答案: ['花', '月'], 意思: '原指四時景色，後比喻浮華空洞的詩文。' },
  { 成語: '三番五次', 缺字: [1, 3], 答案: ['番', '次'], 意思: '形容次數很多。' },
  { 成語: '一字千金', 缺字: [1, 3], 答案: ['字', '金'], 意思: '形容文字價值極高。' },
  { 成語: '畫龍點睛', 缺字: [1, 3], 答案: ['龍', '睛'], 意思: '比喻在關鍵處點明要旨。' },
  // …請幫我補滿至少 10 題，挑國中常見成語
];

每題的「缺字」是索引（0-3），代表要挖空的位置；答案是按缺字順序對應的字。

═══ 遊戲核心玩法 ═══
1. 一場遊戲共 5 題（從題庫隨機抽，不重複）
2. 每題畫面：
   - 上方大字顯示成語，缺字位置用方框「□」代替
   - 例：「一 □ 千 □」
   - 下方排 15 張字卡（隨機排列）：2 個正解 + 13 個干擾字
   - 字卡順序每次都打亂
3. 玩家把字卡用拖曳拉到上方的空格
4. 判定：
   - 拖對 → 字卡變綠、鎖定在空格內、播清脆鈴聲
   - 拖錯 → 字卡彈回原位、紅光閃 0.2 秒
5. 兩個空格都填對 → 顯示「✓ 答對！」+ 成語意思 + 下一題按鈕
6. 5 題完成 → 顯示總成績 + 重新挑戰按鈕

═══ 視覺風格 ═══
- 古色古香、書卷字帖風
- 背景：米黃宣紙色 + 淡淡紋理
- 標題與成語：毛筆楷書字型（用「Noto Serif TC」）
- 空格用紅色方框（像書法字帖的米字格）
- 字卡用淺色木紋或淡黃底 + 黑色楷體
- 答對時空格內字變為紅色（像紅章蓋上去）

═══ 技術需求 ═══
- 單一 index.html（CSS + JS 全部塞進去）
- 不依賴第三方套件（字型用 Google Fonts CDN 可以）
- 中文 UTF-8 編碼
- 拖曳同時支援桌機滑鼠 + 平板觸控

完成後請用本地瀏覽器幫我打開。

請在我目前 IDE 打開的「我的小遊戲」資料夾裡，建一個叫「示範3_打怪RPG」的子資料夾，做一個 index.html。

═══ 題庫資料來源 ═══
請用 fetch 抓這個 Google Sheets 發布的 CSV 網址：

CSV 格式：第一列是表頭「題目,A,B,C,D」，A 欄是正解、B/C/D 是錯誤選項
（如果某欄是「空」就忽略那個選項）

═══ 遊戲核心玩法 ═══
1. 玩家固定一個角色（🧙 巫師），HP 100、初始金幣 0
2. 一場遊戲共 5 個關卡，每關 1 隻怪（依序登場）：
   第 1 關 🐺 野狼（HP 100）
   第 2 關 🦂 毒蠍（HP 100）
   第 3 關 👹 惡鬼（HP 100）
   第 4 關 🐲 巨龍（HP 100）
   第 5 關 👑 魔王（HP 100）
3. 戰鬥流程：
   - 顯示「我方 vs 怪物」+ HP 血條 + 右上角金幣總額

Step 1.1打開 Firebase 控制台

console.firebase.google.com
用你的 Google 帳號登入（沒有就先申請一個 Gmail）。

📋 進到首頁後要做的事

  1. 點中央那張大卡片 「建立 Firebase 專案」
  2. 或點左上角 「+ 新增專案」（兩個結果一樣）
💡 第一次打開可能會看到一張「歡迎使用 Firebase」說明頁，按過去就好。
看到這道「光門」就對了 — 進入 Firebase 世界
Step 1.2輸入專案名稱

📋 操作步驟

  1. 在中央 「專案名稱」欄位輸入名字
    建議用英文小寫，例：my-class-game、trytry、history-quiz
  2. 下方會自動產生一個 ID（例：trytry-9e732）— 不用管它
  3. 勾選 「我接受《Firebase 條款》」
  4. 下面那條「Google 開發人員計畫」保留藍色開啟就好（沒差）
  5. 點右下角藍色 「繼續」按鈕
⚠️ 名字不能用中文，也不能有空白。
專案 ID 一旦建立就不能改，但名字之後可以改。
範例：專案名稱填 TRYTRY，自動產生的 ID 不用管它
Step 1.3關掉 Gemini in Firebase

📋 操作步驟

下一頁標題會是 「Firebase 專案的 AI 輔助功能」

  1. 找到下方 「啟用 Gemini in Firebase」開關
  2. 點它，把藍色關掉變灰色（預設是開的，要手動關！）
  3. 點右下角藍色 「繼續」
❌ 為什麼要關？
我們的 AI 工具用的是 Antigravity / Claude / Gemini CLI，不需要 Firebase 內建的 Gemini。
開了會佔用 quota，沒好處。
把藍色開關「啟用 Gemini in Firebase」點成灰色
Step 1.4關掉 Google Analytics

📋 操作步驟

下一頁標題是 「Google Analytics」

  1. 找最下方 「啟用這項專案的 Google Analytics 功能」開關
  2. 確認是灰色（關閉狀態） — 如果是藍色就點一下把它關掉
  3. 點右下角藍色 「建立專案」
  4. 等 30 秒，看到「Firebase 專案已準備就緒」→ 點 「繼續」
❌ 為什麼要關？
Analytics 是用來分析使用者行為的工具，課程教學用不到，關掉比較單純。
✅ 點「繼續」後會進到專案首頁，第一階段完成！
確認 Analytics 開關是灰色（關閉）

📋 操作步驟

💡 Firestore 是什麼？
就是 Google 的雲端資料庫，類似 MongoDB 那種「文件」結構。
你不用懂內部原理，AI 會幫你寫存取的程式。
點中央白色「建立資料庫」按鈕
Step 2.2選資料庫位置（asia-east1）

📋 操作步驟

⚠️ 位置一經設定就無法變更！
畫面會跳出黃色警告框提醒你。記得選 asia-east1（彰化，台灣），給台灣學生用速度最快、延遲最低。
選錯的話只能砍掉重新建一個專案。
位置一定要選 asia-east1 (Taiwan)，台灣速度最快
Step 2.3用「測試模式」啟動 ⭐

📋 操作步驟

    • 以正式版模式啟動（資料預設不能讀寫，不要選）
  2. 點左下角藍色 「建立」
  3. 等 30 秒，看到資料庫主畫面 ✅
💡 測試模式 vs 正式版模式
· 測試模式：30 天內任何人都能讀寫，30 天後自動鎖死
· 正式版模式：預設誰都不能讀寫

兩個都不是我們要的最終狀態 — 我們選測試模式，下一步馬上把規則改成永久允許。

Step 3把規則改成「永久允許」 ⭐⭐

📋 操作步驟

  1. 點上方 「規則」分頁
  2. 中央會看到一段預設程式碼 → 整段全選刪掉
  3. 貼上下方那段（重點是最後那行 if true）
  4. 點右上角藍色 「發布」按鈕
  5. 看到綠色提示「規則已發布」✅
⚠️ if true 的意思是「任何人都能讀寫」
適合教學、班級遊戲、學生作業這類用途，絕對不要放：
· 學生個資（身分證、生日、地址、家長電話）
· 任何密碼
· 商業機密資料
📋 規則程式碼（整段複製） 
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}
看到第 16 行 allow read, write: if true 就對了
