【學員痛點】
教師花很多時間出考卷／練習題，但學生只是「考完就忘」。
傳統考卷無趣、靜態、無法重複使用、改一題要重做整份。

【課程承諾】
4 小時後，學員能：
1. 把自己的題庫丟進 NotebookLM 自動整理成標準表格

3. 用 4 種小遊戲（選擇題闖關 / 配對 / 轉盤抽題 / 答題轉蛋）
   把同一份題庫變成讓學生「願意一玩再玩」的工具

【核心翻轉點】
讓題庫從「死的 word/PDF」變成「活的 Google Sheets URL」
→ 一份題庫，無限種遊戲玩法
→ 改題目不用改遊戲

【4-1】為什麼用 NotebookLM？  [⏰ 5 分]
─────────────────────────
- NotebookLM 是 Google 出的免費 AI 筆記工具
- 它可以「讀懂」你丟進去的 PDF / Word / 純文字
- 然後幫你整理成你要的格式（表格、摘要、清單）
- 對教師最大價值：把「亂七八糟的題庫檔」變成「乾淨的表格」

[📷 需補圖] NotebookLM 首頁截圖

【4-2】示範：丟一份題庫進 NotebookLM  [⏰ 15 分]
─────────────────────────
[💬 口白] 老師現場操作給學員看，跟著做

步驟：
1. 開 https://notebooklm.google.com/
2. 建立新筆記本 → 上傳題庫檔案（PDF / Word）
3. 在對話框輸入這段 prompt：

【標準 Prompt】
請把這份題庫整理成標準表格，欄位如下：
- 題目（中文題幹）
- A（正確答案）
- B（錯誤選項 1）
- C（錯誤選項 2）
- D（錯誤選項 3）

注意：
- A 欄一律放正確答案，B/C/D 一律放錯誤選項
- 不要包含題號（第 1 題、第 2 題那種）
- 每一列就是一題
- 直接用 Markdown 表格格式輸出，方便我複製到 Google Sheets

[💬 口白] 等它跑完，會看到一張漂亮的表格

【4-3】實作時間 · 學員自己跑一遍  [⏰ 20 分]
─────────────────────────
學員拿出自己準備的題庫檔案，跟著做：
1. 上傳到 NotebookLM
2. 貼上標準 prompt
3. 檢查產出的表格
4. 如果欄位錯誤，就在對話框告訴 NotebookLM：「B 欄應該是錯誤選項，請重新整理」

[💡 老師繞場巡視，幫卡關的學員 debug]
[⚠️ 常見坑]：題庫原本就格式不一致 → NotebookLM 會猜，需要學員人工檢查

【5-2】發布成 CSV / 公開網址  [⏰ 10 分]

[🎯 核心] 這一步是整堂課的「魔法時刻」

步驟：
1. 檔案 → 共用 → 發布到網路

- 你想改題目 → 直接在 Sheets 改
- 改完 → 遊戲自動更新（不用重新部署）
- 想加題目 → 加一列就好

【6-1】概念：fetch 是什麼？  [⏰ 5 分]
─────────────────────────
- 你的網頁 → 對你的 Sheets URL 說：「給我資料」
- Sheets URL → 回你一份 CSV
- 網頁解析 CSV → 變成 JavaScript 物件陣列
- 遊戲拿這個陣列當題目來源

簡單比喻：
- 網頁 = 自動販賣機
- Sheets URL = 後台補貨倉庫
- 老師補貨 → 販賣機自動有新貨

【6-2】示範 prompt  [⏰ 10 分]
─────────────────────────
教學員把這段貼給 Antigravity：

請在桌面建一個「題庫測試」資料夾，做一個 index.html。
需求：
1. 用 fetch 抓取以下 CSV 網址（這是我的 Google Sheets 題庫）：
   [貼你的網址]
2. 解析 CSV，把每一題變成物件：{question, correct, wrongs:[B,C,D]}
3. 隨機選一題顯示在畫面中央
4. 4 個選項按鈕（A/B/C/D 順序要打亂）
5. 點對 → 綠色 + 跳下一題
6. 點錯 → 紅色 + 顯示正確答案
7. 介面要明亮、適合國中生

[💬 口白] 學員會看到自己的題目第一次「跑」在網頁上 → 驚喜時刻

【6-3】常見坑與排錯  [⏰ 10 分]
─────────────────────────
- CORS 錯誤 → Sheets 沒發布成 CSV，重檢查發布設定
- 抓不到資料 → 網址打錯
- 中文亂碼 → fetch 時加 .text() 解碼 UTF-8
- 題目跑出空白 → CSV 第一列不是欄位名
