Visual Web Presentation

一堂課，也可以像故事一樣好看

從零散資料到有節奏、有畫面、有互動的網頁簡報。

先用一句話讓觀眾知道成品會帶來的改變，不急著塞課程目錄。
01 · 先讓內容站穩

觀眾不是來讀資料，是來完成一次理解

01

先有問題

讓學員認出自己正卡在哪裡。

02

再看轉變

把學會前後的差異變得可見。

03

最後能行動

留下一個今天就能做的步驟。

這頁示範三卡片版型。三張卡的句型與長度要一致。
02 · 圖片要有工作

不是每頁放圖，而是每張圖都要推進理解

抽象概念用圖解，人物轉變用情境，章節轉場才用滿版背景。

正式製作時，將此區替換成繪圖模型產生的課程專屬插圖。

圖片生成後放進 assets/images，再以 img 元素取代構圖示意區。
03 · 動態只解釋關係

一次出現一個重點，讓順序自己說話

找到問題

從學員熟悉的卡點開始。

拆出原因

只保留最關鍵的機制。

示範做法

用流程或前後對照呈現。

立刻應用

把知識變成一個小行動。

按下一頁時會先逐個顯示四步，再進到下一張投影片。
04 · 用角色帶出轉折

漫畫不是裝飾，它替觀眾演一次錯誤與改變

資料很多，全部放上去最完整吧？
😵‍💫
先問：這一頁只要觀眾懂什麼？
🧭
一頁一件事，畫面真的會呼吸！
✨
正式課程可把示意表情換成同角色、同畫風的三格 AI 漫畫。長對白保留在漫畫旁的 HTML 對話卡。
05 · 字要放在對的位置

短而重要的字可以被畫出來；精確資訊留給網頁

適合原生設計字

封面短標、章節字、金句、擬聲詞、短對白。

讓繪圖模型直接把字體設計進畫面。

適合 HTML 文字

日期、網址、數據、流程、姓名、長句與必須無誤的資訊。

不要用系統字體後壓到生成圖上。

這是整個 Skill 最重要的圖片文字界線。
Takeaway

好看的簡報，不是裝飾更多；是讓重要的事更快被看見。

用一句可記住的話收尾，再口頭交代下一步。
