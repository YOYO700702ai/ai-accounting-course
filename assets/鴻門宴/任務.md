本文件是交給 Codex 的完整施工規格，已整合所有後續修改：11 場 + 尾聲蒙太奇、實際出圖清單、背景音樂、扣分與 Game Over、統一繁中字體、排座位不洩漏答案、尾聲新增三張圖與多場景切換。

## 一、總體目標

改編自《史記・項羽本紀》「鴻門宴」。玩家以第三人稱旁觀視角，依序經歷：

1. 中軍震怒
2. 項伯夜奔
3. 結親家
4. 項羽動搖
5. 鴻門入帳
6. 三舉玉玦
7. 項莊舞劍
8. 樊噲闖帳
9. 廁所脫身
10. 豎子不足與謀
11. 軍中誅曹
12. 尾聲・歷史鏡頭

學習目標：

- 理解故事完整脈絡：告密 -> 通風報信 -> 拉攏項伯 -> 鴻門赴宴 -> 宴中殺機 -> 樊噲闖帳 -> 劉邦脫身 -> 范增失望 -> 劉邦誅曹 -> 楚漢結局。
- 理解人物性格：項羽重名聲、講義氣但耳根軟；劉邦能屈能伸、果決冷酷；范增清醒但無力；張良冷靜調度；樊噲勇猛救主；項伯重私情。
- 理解古代座位尊卑與人物權力關係。
- 讀懂文言關鍵句與白話對白的對應。

## 四、計分與 Game Over 規則

遊戲採扣分制，不是錯了才看解答而無代價。

常數：

```js
const START_SCORE = 100;
const OPTION_PENALTY = 30;
const SEAT_PENALTY = 20;
```

規則：

- 起始分數：100。
- 選項/題目答錯：扣 30。
- 排座位放錯人物：扣 20。
- 座位區已滿仍放入：扣 20。
- 分數扣到 0：立即進入 `gameover.png` 畫面。
- Game Over 畫面可點擊重新開始。
- 重新開始後：
  - 分數回 100。
  - 回封面。
  - 清空所有劇情、選項、座位狀態。

UI：

- 左上顯示 `分數 100`。
- 扣分時顯示短暫 toast，例如 `選項錯誤 -30，剩 70`。
- 分數 panel 有短暫 hit 動畫。

## 五、劇情資料結構

劇情集中於 `script.js` 的 `acts` 陣列。每場格式：

```js
{
  title: "第一場・中軍震怒",
  scene: "scene1.png",
  events: [
    { type: "line", character: "narrator", text: "..." },
    { type: "line", character: "xiangyu", quote: "...", text: "..." },
    { type: "question", label: "...", character: "...", prompt: "...", options: [...], correct: 0, explanation: "..." },
    { type: "seat" }
  ]
}
```

事件型別：

- `line`：一般對白或旁白。
- `question`：三選一互動題。
- `seat`：排座位小遊戲。

`line` 可加：

- `quote`：文言金句，顯示在對話框上層。
- `inner: true`：內心 OS，用淡灰斜體呈現。
- `scene`：單句切換背景圖，尾聲蒙太奇使用。

例如：

```js
{ type: "line", character: "liubang", inner: true, text: "曹無傷，你死定了。" }
{ type: "line", character: "narrator", scene: "scene_wujiang.png", text: "四年後，他在垓下圍住了項羽。烏江邊，項羽自刎。" }
```

## 八、座位排列遊戲

### 8-1 史實座次

| 人物 | 座位 |
|---|---|
| 項羽 | 東向坐・最尊 |
| 項伯 | 東向坐・最尊 |
| 范增 | 南向坐・次尊 |
| 劉邦 | 北向坐・較卑 |
| 張良 | 西向侍・陪侍 |

東向位可坐 2 人，所以遊戲是四個位置、五個人物。

### 8-2 玩法

- 背景使用 `scene3.png`。
- 畫面上標出四個方位區塊。
- 提供五張人物牌。
- 支援點選人物 -> 點座位。
- 支援桌機拖曳。
- 全部正確才可繼續。
- 正確排完後「繼續」按鈕立即可用；人物介紹可看但不是強制。

### 8-3 不可洩漏答案

排座位過程不能直接告訴玩家答案。

禁止：

- 點人物時顯示「項羽是宴席主人，坐最尊東向位」。
- 放錯時顯示「正確位置是北向」。
- 開場提示直接寫「項羽與項伯同坐」。

應顯示：

- 開場：`請把五個人物排入四個方位。注意：有一個方位可以坐 2 人。`
- 點人物：`已選 劉邦。請點選你判斷的座位。`
- 放錯：`劉邦 不能放在東向坐・最尊。請重新判斷座次。`
- 座位已滿：`東向坐・最尊・2席 已滿，請改放其他座位。`

扣分：

- 放錯人物：扣 20。
- 放到已滿座位：扣 20。
- 錯誤後人物牌不可留在錯誤座位，必須留在/回到手牌區。

### 8-4 過關後介紹

只有座次全對後，才可點擊人物看介紹。這時可以揭露座次原因。

人物介紹：

- 項羽：西楚霸王，兵力四十萬，宴席的主人。坐最尊的東向位，盡顯其主導地位與驕傲自負。
- 項伯：項羽的叔父，卻因私交暗中保護劉邦。與項羽同坐尊位，身分使然。
- 范增：項羽的謀士，被尊稱「亞父」。坐次尊位，是宴上最想殺劉邦的人。
- 劉邦：此時兵力僅十萬，以謝罪者身分赴宴，只能坐較卑的北向位，反映其弱勢與隱忍。
- 張良：劉邦的謀士，以隨從身分「侍坐」，坐最卑的西向位，全程冷靜應變、護主脫險。

總結：

古人宴席「東向為尊」。項羽讓自己與叔父坐最尊位、范增次之，卻把客人劉邦安排在較卑的北向，張良更只能陪侍。這個座次把雙方的高下、項羽的傲慢與輕敵，全寫在空間裡。

## 九、介面與排版

### 9-1 對話

- 對話框在下方。
- 左側放角色立繪，立繪要夠大，能突出並占據左側畫面。
- 右側放：
  - 角色名
  - quote 文言金句
  - text 白話對白
  - 繼續按鈕
- 對話文字要大，適合投影與學生閱讀。
- 選項不得遮住題目文字。

### 9-2 題目選項

- 題目文字在上方對話框。
- 選項卡片在下方/右側，不遮住題目。
- 選項卡片要夠大，文字清楚。
- 選後畫面不要讓未選狀態文字和解說重疊。

### 9-3 立繪

- 立繪不應太小。
- 一般對話與題目模式都要讓角色在左側有存在感。

### 9-4 字體

- 全站統一繁中字體。
- 不使用會造成繁中人名字形不一致的毛筆字體。
- 避免「項伯」「項羽」「范增」「劉邦」等名字出現異體或簡體風格。
