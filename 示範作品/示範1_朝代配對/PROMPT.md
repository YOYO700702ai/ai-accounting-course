# 示範 1：朝代配對遊戲 · Prompt

> 把下面這整段複製貼到 Antigravity 的 Claude / Codex 對話框，
> AI 就會幫你做出一模一樣的配對遊戲。
> 這個範例**不需要 CSV 題庫**，題目直接寫死在程式裡，最快可以看到成果。

---

```
我是國中歷史老師，要做一個「朝代配對」練習網頁給學生玩。
請在我目前 IDE 打開的「我的小遊戲」資料夾裡，建一個叫「示範1_朝代配對」的子資料夾，做一個 index.html。

═══ 題目資料（直接寫在 JS 裡，不用外部檔案）═══
請在 JS 裡建兩個陣列：
- 6 個朝代（每個朝代有 id 跟名稱）
- 15+ 個事件（每個事件有 dynasty_id 對應到朝代，可以多對一）
- 每個朝代可以對應 1 個以上的事件（數量不一致沒關係）

朝代陣列（左欄）：
  { id: 1, name: "秦" }
  { id: 2, name: "漢" }
  { id: 3, name: "唐" }
  { id: 4, name: "宋" }
  { id: 5, name: "明" }
  { id: 6, name: "清" }

事件陣列（右欄，至少 15 個）：
  { id: 'e1',  dynasty_id: 1, text: "統一六國",     icon: "⚔️" }
  { id: 'e2',  dynasty_id: 1, text: "修築萬里長城", icon: "🧱" }
  { id: 'e3',  dynasty_id: 1, text: "焚書坑儒",     icon: "🔥" }
  { id: 'e4',  dynasty_id: 2, text: "張騫通西域",   icon: "🐪" }
  { id: 'e5',  dynasty_id: 2, text: "文景之治",     icon: "📜" }
  { id: 'e6',  dynasty_id: 2, text: "罷黜百家獨尊儒術", icon: "📖" }
  { id: 'e7',  dynasty_id: 3, text: "貞觀之治",     icon: "👑" }
  { id: 'e8',  dynasty_id: 3, text: "安史之亂",     icon: "💥" }
  { id: 'e9',  dynasty_id: 3, text: "玄奘西行取經", icon: "🧘" }
  { id: 'e10', dynasty_id: 4, text: "王安石變法",   icon: "📜" }
  { id: 'e11', dynasty_id: 4, text: "岳飛抗金",     icon: "🗡️" }
  { id: 'e12', dynasty_id: 5, text: "鄭和下西洋",   icon: "⛵" }
  { id: 'e13', dynasty_id: 5, text: "永樂大典編成", icon: "📚" }
  { id: 'e14', dynasty_id: 6, text: "鴉片戰爭",     icon: "💣" }
  { id: 'e15', dynasty_id: 6, text: "康熙盛世",     icon: "🌅" }
  { id: 'e16', dynasty_id: 6, text: "辛亥革命終結", icon: "🏛️" }

═══ 遊戲核心玩法 ═══
1. 畫面分左右兩區
   - 左欄：6 個朝代「卡格」（每格上方寫朝代名，下方是「掉落區」，可以放多張事件卡）
   - 右欄：所有事件卡（15+ 張），順序隨機打亂顯示
2. 玩家從右欄把事件卡「拖曳」到左欄對應的朝代格子裡
3. 用 HTML5 drag-and-drop API（dragstart / dragover / drop）
4. 配對判斷：
   - 正確（dynasty_id 相符）→ 事件卡變綠 + 短暫發光，鎖定在朝代格內（堆疊顯示）
   - 錯誤（dynasty_id 不符）→ 紅色閃光 0.3 秒 + 卡片自動彈回右欄原位
5. 每次嘗試（不論對錯）都計數一次「嘗試次數」
6. 右欄的事件卡全部正確放完 → 切換到「過關畫面」：
   - 金色閃光動畫
   - 顯示「全部過關！」「總共嘗試 X 次」「答對率 15/X」
   - 「再玩一次」按鈕（重新打亂、清空）

═══ 視覺風格 ═══
- 古典中國風：深色底（深棕 #2b1810 或墨黑）+ 金色 (#d4af37) 邊框 + 紅色 (#a8201a) 點綴
- 朝代卡：金色邊框、米黃底、朝代名用大字（楷書或宋體感）
- 事件卡：紅色邊框、深色底、icon 大字 + 事件名稱
- 朝代格內已配對成功的事件卡用較小的縮圖形式堆疊顯示（避免太擠）
- 字型用「Noto Serif TC」之類有古典感的襯線字
- RWD：手機上改成上下兩欄；事件卡區可橫向捲動

═══ 技術需求 ═══
- 單一 index.html（CSS + JS 全部塞在一個檔案裡，不要拆檔）
- 不依賴任何第三方套件、不用框架
- 中文 UTF-8 編碼
- 題目資料直接寫在 JS 裡，不需要外部 CSV / JSON
- 用原生 HTML5 drag-and-drop（手機版若不支援可降級成「點朝代 → 點事件」也可以）
- 朝代格的「掉落區」要能容納多張事件卡（堆疊或排列）

完成後請用本地瀏覽器幫我打開。
```

## 📝 改造小撇步

裝出來後，可以叫 AI 再改：

- **加更多事件**：「再幫我各朝代多加 2-3 個事件，總數 30 張」
- **換主題**：「改成『國家 vs 名勝古蹟』，一個國家配多個古蹟」「改成『動物 vs 棲息地』」
- **加計時**：「右上角加倒數 90 秒，時間到沒配完算失敗」「記錄最快通關時間，存 localStorage」
- **改視覺**：「改成日式和風（櫻花粉 + 木質感）」「每個朝代格加一張代表圖示」
- **加難度**：「加入 5 張干擾卡（沒有對應任何朝代）」「配對成功後彈出該事件的一句冷知識教學」
- **改互動**：「改成翻牌記憶配對」「加音效（正確叮一聲、錯誤蹦一聲）」
